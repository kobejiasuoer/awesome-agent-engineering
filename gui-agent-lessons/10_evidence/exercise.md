## 练习 1：跨页对比取证（方法练习）

硬任务「对比 LangGraph 与 CrewAI 最近 release」需要跨两个项目的页取证。请：

1. 用 `deep_browse` 分别从 LangGraph 和 CrewAI 的列表页出发，各取一条证据链。
2. 把两条链合并，让 writer 做对比（谁更新/版本号差异/变更方向）。
3. 报告里对比结论各带两个项目的 URL+时间。

**验收**：报告有「LangGraph v0.12.0（来源+时间）vs CrewAI v0.x（来源+时间）」的对比结构，每个结论可回访。

## 练习 2：快照落盘存档（设计实验类）

本课快照存在 Evidence.snapshot 字段里（内存）。真实场景要落盘——即使原页改版/下线，快照仍在。设计：

1. **假设**：落盘快照让证据在网站改版后仍可审计。
2. **实验**：
   - 给 BrowserTool 加 `snapshot_dir` 配置，deep_browse 每步把快照写 `{url_hash}.txt`。
   - 改 `to_citation` 附快照文件路径（如「快照见 snapshots/abc123.txt」）。
   - 模拟原页改版（改 test_pages），验证快照仍是旧内容。
3. **预期**：改版后快照不变，引用仍可审计。

**验收**：快照文件落盘，改版后快照内容不变。这是「来源可回访」的终极保障——连原页没了也不怕。

## 练习 3：证据冲突检测（理解类）

多步取证可能拿到冲突证据（A 页说 v0.12，B 页说 v0.11）。回答：

1. 怎么检测证据冲突？（提示：对比同主题不同证据的关键字段）
2. 冲突时采信谁？（提示：访问时间新的优先 / 多数优先 / 标注冲突让 writer 说明）
3. 这和 frontier L05 的「事实冲突检测」是同构问题吗？

**验收**：能说出证据冲突检测的规则，并指出它和 frontier L05 reviewer 的事实通道同构——都是「新旧信息矛盾时怎么处理」。本课是跨页证据冲突，frontier 是跨会话记忆冲突。

## 练习 4：思考题——证据链 vs RAG 引用（取舍类）

RAG 也有「引用来源」（rag-lessons 的防幻觉引用）。本课的证据链和 RAG 引用有什么同异？回答：

1. RAG 引用的是什么？（检索到的文档片段 + 出处）
2. 证据链引用的是什么？（浏览提取的内容 + URL + 访问时间 + 快照）
3. 为什么 GUI agent 的证据链多带了「访问时间」和「快照」？（提示：RAG 文档库静态，网页动态会变）

**验收**：能说出「RAG 引用面向静态文档库（不变），证据链面向动态网页（会变）——所以证据链多带时间戳和快照来对抗网页的易变性」。这是 GUI agent 证据链相对 RAG 引用的独有设计。
