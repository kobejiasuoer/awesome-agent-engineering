# Security Policy

## Reporting a vulnerability

请不要通过公开 Issue 报告可能泄露 API Key、绕过鉴权、执行任意代码或触发敏感浏览器动作的漏洞。

请使用 GitHub 的 [Private vulnerability reporting](https://github.com/kobejiasuoer/awesome-agent-engineering/security/advisories/new)，并提供：

- 受影响的课程或作品项目
- 最小复现步骤和预期影响
- 已测试的版本、操作系统与 Python 版本
- 可行的修复建议（如有）

维护者会先确认报告，再协调修复与披露时间。仓库中的 Agent、代码解释器和浏览器控制示例用于教学；部署到真实环境前，仍需根据你的威胁模型配置沙箱、网络策略、密钥管理和人工确认边界。
