# 技能：对比表生成流程

> 适用场景：研究主题涉及"A vs B vs C"对比时，产出结构化对比表。

## 触发条件
当研究摘要中出现以下信号时启用：
- "对比"、"比较"、"vs"、"区别"、"差异"
- 多个并列对象（如"Python/TypeScript/Java"）
- "优缺点"、"取舍"

## 对比表格式

```
| 维度 | 对象A | 对象B | 对象C |
|------|-------|-------|-------|
| 维度1 | 值 | 值 | 值 |
| 维度2 | 值 | 值 | 值 |
```

## 流程
1. 从 findings 中识别对比对象（通常是子问题里的并列项）
2. 提取对比维度（从发现中归纳，如"性能/易用性/生态/成本"）
3. 填充每个对象在每个维度上的值
4. 维度不足 3 个时，不生成表格（改用文字描述）
5. 表格后附一句"对比结论"（不是罗列数据，要有判断）

## 注意
- 表格用于**结构化对比**，不是所有内容都塞表格
- 观点综述仍用文字，只有"可量化维度"才进表格
- 缺数据时写"暂无"，不要编
