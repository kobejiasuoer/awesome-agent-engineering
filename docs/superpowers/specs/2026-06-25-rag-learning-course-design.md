# RAG 渐进式学习课程设计

## 背景

学习者画像：
- **技术基础**：会 Python，但没接触过大模型（LLM）相关开发
- **学习目标**：系统理解 RAG 原理（不只是调包，要懂 why 和取舍）
- **技术选型**：智谱 GLM（embedding-3 + GLM-4）+ Chroma 本地向量库
- **学习形式**：渐进式课程，由浅入深

工作区从零开始，本课程既是学习材料，也是一个可运行、可改的实验场。

## 设计原则

1. **第一课就出成果**：避免纯理论劝退，20 分钟内跑通一个完整 RAG。
2. **课时顺序 = RAG 真实数据流**：学完对整个系统有完整心智模型，每课对应 RAG 的一道关卡。
3. **每课三件套**：原理讲解（Markdown，讲 why 和取舍）+ 可运行代码（带详细中文注释）+ 小练习。
4. **原理优先于框架**：关键环节先用朴素方式手写一遍（如暴力检索），再用框架（如 Chroma），让学习者看到"框架替你做了什么"。
5. **可视化为先**：能画图、能打印对比表的地方，都做出来，降低理解门槛。

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 大模型 | 智谱 GLM-4 | 国内可用、中文友好 |
| Embedding | 智谱 embedding-3 | 与 GLM 配套，2048 维 |
| 向量库 | Chroma | 本地运行、免安装服务、API 简单 |
| 框架 | 原生 SDK 为主，LangChain 辅助 | 原生 SDK 让原理更透明；LangChain 在工程化阶段引入 |
| Reranker | 智谱或本地 cross-encoder | 进阶课程使用 |
| 评估 | RAGAS | 业界主流 RAG 评估框架 |

## 课时设计（共 9 节）

### Lesson 01 — 先跑通：你的第一个 RAG
**目标**：20 分钟内跑通完整 RAG，建立全局认知。

**原理 README**：
- RAG 解决的三大问题：幻觉、知识过期、私有数据
- 整条流水线全景图：`问题 → embedding → 向量检索 → 拼 prompt → 生成`
- 为什么需要"检索"这一步（对比直接问 GLM）
- 代码里每一行对应流水线的哪个环节

**code.py**：
- 硬编码 3-5 条知识文本
- 用 embedding-3 向量化存入 Chroma
- 接收问题 → 检索 top-k → 拼 prompt 调 GLM-4
- 打印检索片段 + 最终回答（让学习者看见"模型看到了什么"）

**练习**：改一条知识、问文档里没有的问题，观察模型表现。

---

### Lesson 02 — 深入 Embedding：向量如何表示语义
**目标**：理解 embedding 不是黑盒，建立"向量空间 = 语义空间"的直觉。

**原理 README**：
- 文本如何变成一串数字（token → 向量）
- 为什么相似文本向量也相似（几何直觉 + 余弦相似度公式）
- 维度的含义（embedding-3 默认 2048 维，可降维）
- 核心认知：embedding 质量直接决定 RAG 上限

**code.py**：
- 把 6-8 个语义相近/相反的句子向量化
- 计算两两余弦相似度，打印相似度矩阵
- 可视化（matplotlib 或纯文本降维图）展示"相似句子聚成一簇"

**练习**：造一组句子，预测哪些相似，用代码验证。

---

### Lesson 03 — 向量检索：从相似到 Top-K
**目标**：搞懂检索内部发生了什么，以及为什么需要向量数据库。

**原理 README**：
- 暴力计算 vs 近似最近邻（ANN）的取舍
- 余弦相似度 / 欧氏距离 / 点积的区别和适用场景
- Chroma 是什么、为什么 RAG 需要专门的向量库
- collection / metadata / distance 概念
- Top-K 的 K 怎么选

**code.py**：
- 构建小型 Chroma 集合（持久化到本地磁盘）
- 对比"暴力计算"和"Chroma 检索"的结果与速度
- 演示 metadata 过滤（按来源/类别筛选）
- 展示不同 distance 度量下的排序差异

**练习**：调整 K 值，观察返回结果质量变化。

---

### Lesson 04 — 文档处理：加载与切块 (Chunking)
**目标**：理解为什么不能把整篇文档直接扔进去，切块是影响极大的环节。

**原理 README**：
- 为什么必须切块（上下文窗口、检索精度、成本）
- 切块策略对比：固定长度 / 按字符 / 按语义(RecursiveCharacter) / 按结构(标题段落)
- 关键参数：chunk_size 和 chunk_overlap 的取舍
- 不同文档类型（文本/Markdown/PDF/代码）的差异
- 表格、图片等结构化内容的处理思路

**code.py**：
- 加载真实 Markdown 文档
- 用不同 chunk_size (200/500/1000) 切，对比结果
- 演示 RecursiveCharacterTextSplitter 按分隔符优先级智能切
- 存入 Chroma 供后续课程使用

**练习**：用自己的文档试不同参数，挑出最合理配置。

---

### Lesson 05 — Prompt 工程：把检索结果喂给大模型
**目标**：检索到了素材，怎么组织 prompt 让模型答得又准又稳。

**原理 README**：
- RAG prompt 标准结构：`指令 + 检索上下文 + 用户问题`
- 关键设计：明确要求"只依据材料回答，没有就说不知道"——压低幻觉的核心
- few-shot 示例的作用
- 上下文窗口管理：检索片段太多怎么办（截断/压缩/Map-Reduce）
- 引用溯源思路

**code.py**：
- 用前面建好的向量库检索
- 对比"无约束 prompt" vs "防幻觉 prompt" 的回答差异（同一问题看模型变老实）
- 实现带引用溯源的回答
- 演示流式输出

**练习**：设计文档里没有的问题，看防幻觉指令的效果。

---

### Lesson 06 — 进阶检索：混合检索 + Rerank
**目标**：RAG 进阶分水岭，理解为什么单纯向量检索不够。

**原理 README**：
- 向量检索的局限：对关键词/专有名词/编号不敏感
- BM25 关键词检索原理与优势
- 混合检索 (Hybrid)：向量 + BM25 加权融合
- Rerank 重排序：cross-encoder 召回+精排两段式架构
- 召回 vs 精排：工业界标准

**code.py**：
- 构造"向量检索翻车"场景（带编号/专有名词的查询）
- 用 rank-bm25 实现关键词检索
- 实现混合检索（加权融合）
- 接入 reranker，演示两段式
- 对比四种方式的检索质量

**练习**：调整融合权重，观察结果变化。

---

### Lesson 07 — Query 改写：让检索更准
**目标**：用户问题表达不清导致检索不准，学会改造 query。

**原理 README**：
- 为什么原始问题往往不好直接检索（太短/口语化/指代不明）
- HyDE：让模型先编"假设答案"，用假设答案的向量去检索
- 多查询展开：把一个问题展开成多个子问题分别检索
- 查询路由：根据问题类型分发到不同检索策略

**code.py**：
- 实现 HyDE（LLM 生成假设答案 → 向量检索）
- 实现多查询展开（LLM 生成多个改写 → 合并结果）
- 对比"原始 query 检索" vs "改写后检索"的命中率

**练习**：找几个口语化问题，对比改写前后效果。

---

### Lesson 08 — RAG 评估：怎么知道好不好
**目标**：用数据客观衡量 RAG 质量，不凭感觉。

**原理 README**：
- 为什么需要评估（RAG 链路长，哪里出问题难定位）
- RAGAS 三维指标：检索相关性 / 答案忠实度(faithfulness) / 答案相关性
- 构造评估数据集的思路
- 如何根据指标定位问题（检索 vs 生成）

**code.py**：
- 构造一个小评估集（问题 + 标准答案 + 参考文档）
- 跑 RAGAS 得到三维分数
- 对比不同参数（如不同 chunk_size）下的分数变化

**练习**：对前面课程的某个 RAG 配置做评估，写结论。

---

### Lesson 09 — 工程化：接近生产可用
**目标**：把前面学的拼成一个完整小系统，了解工程化要点。

**原理 README**：
- 流式输出的工程意义
- 引用溯源的落地实现
- 多文档管理（不同来源、权限、更新）
- 缓存优化（embedding 缓存、查询缓存）
- RAG + Agent 结合的思路（工具调用）

**code.py**：
- 实现一个带引用溯源的命令行问答程序
- 加入 embedding 缓存避免重复计算
- 支持多文档目录
- 演示流式 + 引用的完整体验

**练习**：接入自己的文档集，做一个能用的问答助手。

## 实施说明

- **分批制作**：不一次性写完 9 课。先完成 Lesson 01，让学习者跑通、确认体验 OK，再逐课推进。这样能根据真实反馈调整后面课程。
- **第一课交付物**：`README.md` + `code.py` + `exercise.md` + `requirements.txt` + `.env.example` + `data/sample_docs/` + 顶层 `README.md`。
- **环境准备**：Python 3.9+，安装 requirements.txt，配置 `.env` 里的智谱 API Key。
- **不做的事**：不引入 Docker、不部署服务、不做前端界面——保持学习聚焦在 RAG 本身。
